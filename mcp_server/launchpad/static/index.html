About Chromap Launchpad

        

          Chromap Launchpad is a browser helper that talks to the same workflow definitions as
          Chromap MCP. Use Copy to paste the command into your
          terminal, or Run in shell (localhost only) to start the same command
          on the server host in the background.
        

        

          Load parameters / Save parameters read or write a
          JSON file via the browser file picker (client-side only). Format:
          { "workflow_id": "…", "params": { … } }
          or a flat object of parameter names to values.
        

        

          Quit server stops the Chromap Server (Launchpad + MCP) running on this host.
          Shortcut: Ctrl+Shift+Q. Localhost only (for safety).
        

        

          Validate and Generate command default to schema and type
          checks only. Enable Check paths on server host when you want Launchpad to
          verify file and directory existence on the machine running Chromap Server. Run in shell
          always does that stricter server-host validation before launch.
        

        

          By default the recipe list shows only basic Chromap CLI workflows
          (chromap_*). Enable Include test & other recipes for
          UCSF production, E2E tests, and the rest (private entries appear on localhost).
        

        

          Chromap Server hosts Chromap MCP (for agents) and this UI on one port.
        

        

          Script Lane (second tab) translates annotated Bash to Workflow IR via
          bwb-nextflow-utils, checks simple-local viability, and can execute locally on the
          server host when Launchpad is opened over localhost. It never uses Run in shell.
        

        

          Composition (third tab) lists curated constructor profiles from
          bwb-nextflow-utils, shows backend-computed recipe inputs (kind-driven
          fields), and builds execution-recipe drafts plus review artifacts (normalized recipe, preview,
          script, manifest). It does not run composed workflows; execution stays out of band.
